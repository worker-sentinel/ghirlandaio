# Koneksi Jaringan (Wi-Fi)

> Jika instalasi menggunakan jaringan Wi-Fi, gunakan utilitas iwd:

```
iwctl
```
> Memindai Wi-Fi sekitar
```
station wlan0 get-networks
```
> Menghubungkan ke jaringan wifi dan memasukkan password
```
station wlan0 connect "NamaWifi"
# Masukkan password Wi-Fi Anda, lalu:
exit
```

> Pastikan service iwd sudah berjalan sebelum menggunakan iwctl.

> Gunakan lsblk atau cfdisk untuk melihat dan mengatur partisi disk yang tersedia.

```
lsblk
```

# Enkripsi Partisi Menggunakan LUKS

> Format partisi Linux menggunakan LUKS.

## Buat container LUKS
```
cryptsetup luksOpen /dev/nvme0n1p6 lit
```

> Konfirmasi dengan mengetik YES (huruf kapital), lalu masukkan dan verifikasi passphrase.

## Buka container LUKS
```
cryptsetup luksOpen /dev/nvme0n1p6 lit
```

> Ganti lit dengan nama mapping sesuai keinginan Anda

> Selanjutnya buka partisi terenkripsi.

> Perintah cryptsetup luksFormat digunakan untuk menginisialisasi partisi sebagai media penyimpanan terenkripsi menggunakan standar LUKS (Linux Unified Key Setup). Seluruh data yang tersimpan pada partisi nantinya akan dilindungi menggunakan passphrase sehingga tidak dapat diakses tanpa proses autentikasi.
Saat menjalankan perintah ini, sistem akan meminta konfirmasi dengan mengetik *YES* (huruf kapital), kemudian meminta pengguna memasukkan passphrase sebanyak dua kali.

# Buat Volume Group & Logical Volumes
```
vgcreate server /dev/mapper/lit

lvcreate -L 8G   server -n root
lvcreate -L 5G   server -n vars
lvcreate -L 1G   server -n vlog
lvcreate -L 1G   server -n vaud
lvcreate -L 1.5G server -n vtmp
lvcreate -l50%FREE  server -n home
lvcreate -l100%FREE server -n podman
```

# Membuat Filesystem & Mounting
> Format setiap logical volume dengan ext4

```
mkfs.ext4 /dev/server/root
mkfs.ext4 /dev/server/vars
mkfs.ext4 /dev/server/vlog
mkfs.ext4 /dev/server/vaud
mkfs.ext4 /dev/server/vtmp
mkfs.ext4 /dev/server/home
mkfs.ext4 /dev/server/podman
```

## Mount root terlebih dahulu
```
mount /dev/server/root /mnt
```

## Mount partisi boot (EFI)
```
mount --mkdir -o uid=0,gid=0,fmask=0077,dmask=0077 /dev/nvme0n1p5 /mnt/boot
```

## Mount direktori sistem lainnya
```
mount --mkdir -o rw,nodev,nosuid,relatime              /dev/server/vars   /mnt/var
mount --mkdir -o rw,nodev,nosuid,noexec,relatime        /dev/server/vlog   /mnt/var/log
mount --mkdir -o rw,nodev,nosuid,noexec,relatime        /dev/server/vaud   /mnt/var/log/audit
mount --mkdir -o rw,nodev,nosuid,noexec,relatime        /dev/server/vtmp   /mnt/var/tmp
mount --mkdir -o rw,nodev,nosuid,relatime               /dev/server/home   /mnt/home
mount --mkdir -o rw,nodev,nosuid,relatime               /dev/server/podman /mnt/var/lib/containers
```

## Instalasi Sistem Dasar (pacstrap)
```
pacstrap /mnt base amd-ucode linux-lts linux-lts-headers linux-firmware \
  mkinitcpio lvm2 git neovim firewalld openssh sudo pacman wget curl grep iwd podman
```

> Pastikan nama paket benar — typo seperti mkinitcipio akan menyebabkan error target not found.

# Masuk ke chroot
```
arch-chroot /mnt
```
# Konfigurasi mkinitcpio preset
> Edit file preset mkinitcpio sesuai kernel yang digumakan:
```
nvim /etc/mkinitcpio.d/linux-lts.preset
```
``` 
 mkinitcpio preset file for the 'linux-lts' package

ALL_config="/etc/mkinitcpio.conf"
ALL_kver="/boot/vmlinuz-linux-lts"
ALL_kerneldest="/boot/vmlinuz-linux-lts"

PRESETS=('default')
```

>  UKI output path
default_uki="/boot/EFI/Linux/arch-linux-lts.efi"

> Pastikan path default_uki mengarah ke direktori yang writable dan sudah di-mount dengan benar.

# Mengaktifkan Service Dasar
```

systemctl enable iwd

```

```
exit
umount -R /mnt
```
