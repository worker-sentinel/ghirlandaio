## PEMBAHASAN
### Pengertian Source Code
<p align="justify">
&nbsp;&nbsp;&nbsp;&nbsp;Dilansir dari TechTarget, source code (kode sumber) adalah komponen dasar dari sebuah program komputer yang dibuat oleh seorang pemrogram yang biasanya ditulis dalam bentuk fungsi, deskripsi, definisi, panggilan, metode, dan pernyataan operasional lainnya. Source code dibuat agar dapat dibaca dan dikembang dengan cara yang dapat dipahami oleh pengembang teknologi maupun pengguna lainnya.
</p>
<br>

### Lisensi Source Code
<p align="justify">
&nbsp;&nbsp;&nbsp;&nbsp;Masih melansir dari laman yang sama, source code bisa bersifat terbuka atau kepemilikan dan keduanya tentu memiliki perbedaan. Sebagai contoh, seorang pengguna memasang perangkat lunak seperti Microsoft Office yang memiliki source code yang bersifat kepemilikan sehingga pihak Microsoft hanya memberikan akses saja kepada pelanggan sementara seseorang yang menggunakan Apace OpenOffice memiliki source code yang bersifat terbuka sehingga dapat diunduh dan dimodifikasi. 
</p>
&nbsp;&nbsp;&nbsp;&nbsp;Pihak Microsoft memiliki source code yang bersifat kepemilikikan karena untuk melindungi kekayaan intelektual dan untuk mencegah pelanggan dari mengubah source code yang dapat merusak program. Sementara open source seperti contoh pada Apache OpenOffice dirancang dengan gagasan source code harus tersedia karena sebagai bentuk kolaboratif pengembang untuk mengembangkan perangkat lunak yang dimiliki untuk membuatnya lebih kuat dan aman. Pengguna bebas mengambil open source di bawah lisensi publik seperti Lisensi Publik Umum GNU.
</p>
<br>

### Tujuan Source Code
<p align="justify">
&nbsp;&nbsp;&nbsp;&nbsp;Source code memiliki tujuan sebagai dasar untuk pembuatan perangkat lunak, dan memudahkan penyesuaian instalasi perangkat lunak bagi pengguna yang memiliki akses ke source code. Pengembang lain dapat menggunakan source code untuk membuat program serupa untuk platform operasi lain. Akses ke source code juga memungkinkan pemrogram untuk berkontribusi pada komunitas mereka melalui berbagi kode untuk tujuan pembelajaran atau mendaur ulang bagiannya untuk aplikasi lain.
</p>
<br>

### Sejarah Source Code
<p align="justify">
&nbsp;&nbsp;&nbsp;&nbsp;Perangkat lunak yang pertama ditulis dalam kode biner pada tahun 1840-an. Program semacam itu kemungkinan merupakan contoh awal dari dari adanya source code, namun sebenarnya itu juga tergantung pada sudut pandang seseorang. Salah satu contoh paling awal dari source code ditulis oleh Tom Kilburn, pelopor awal dalam ilmu komputer. Kilburn menciptakan sebuah program digital pertama yang berhasil disimpan secara elektronik dalam memori komputer pada tahun 1948.
</p>
&nbsp;&nbsp;&nbsp;&nbsp;Di tahun 1950-an dan 1960-an, source code biasanya disediakan secara gratis dengan perangkat lunak oleh perusahaan yang membuat program. Source code manjadi lebih produktif dan dicari ketika perusahaan komputer yang berkembang memperluas penggunaan perangkat lunak. Sebuah majalah komputasi biasanya mencetak source code pada halaman mereka dengan pembaca perlu mengetik ulang kode untuk penggunaan mereka sendiri. Kemudian floppy disk menurunkan harga untuk berbagi source code secara elektronik.
</p>
<br>

## KESIMPULAN
<p align="justify">
&nbsp;&nbsp;&nbsp;&nbsp;Source code adalah sebuah program dalam komputer yang bisa digunakan oleh penyedia teknologi untuk dapat memperbaiki jika ada masalah sekaligus juga untuk dapat mengembangkannya. Source code bertujuan sebagai dasar pembuatan perangkat lunak dan memudahkan memudahkan penyesuaian instalasi perangkat lunak bagi pengguna yang memiliki akses ke source code. Source code dapat bersifat terbuka, yang artinya pengguna lain selain bisa mengunduh suatu program bisa pula memodifikasinya. Sementara source code yang bersifat kepemilikan membuat pengguna hanya bisa mengaksesnya tanpa bisa memodifikasinya. Program digital yang dibayt oleh Tom Kilburn pada tahun 1948 menjadi salah satu contoh awal source code karena Kilburn berhasil menyimpan program digital tersebut secara elektronik dalam memori komputer.
</p>
