# Bagaimana Sikap Anda Mengenai Closed Source

## Latar Belakang
Closed source adalah software yang source codenya tidak dibuka untuk umum. Sang pemilik code yang closed source bisa membagi source codenya melalui lisensi, entah dengan gratis maupun membayar.

Meskipun gratis, lisensi tertentu bisa membuat software tidak sepenuhnya open source. Misalnya jika di lisensi tersebut ada larangan untuk memodifikasi code, maka software ini tidak open source.

Seperti adanya lisensi yang mengharuskan pengguna untuk menyediakan dana/financial, ada pun kekurangan dari closed source yaitu celah yang terbuka bisa dimanfaatkan untuk pengambilan informasi, dan tidak ada support khusus.

Metode pengumpuluan data dilakukan dengan mencari referensi, yaitu:
Arsito Ari Kuncoro S.Kom, M.Kom - Closed Source

## Hasil
Sikap saya mengenai closed source adalah saya menentang closed source karena menjadi kurang transparan dan akses terbatas sehingga kita tidak bisa mengutak-atik program secara bebas dan mengembangkannya, jadi terdengar kikir dan kapitalis.

## Kesimpulan
Saya kontra dengan closed source karena walaupun closed source punya sisi baik, tapi sisi baik itu biasanya hanya dinikmati oleh pihak internal, jadi closed source sendiri adalah bentuk nyata monopoli kapital, pengguna tidak bisa mengakses kode karena dikunci  oleh korporasi dan jadi tidak bisa mengembangkannya.
